# Garvis PoC Demo Runbook (10-15 Minutes)

**Goal:** Showcase an auditable, truth-driven AI governance system with provenance and explainability.

**Audience:** Investors, collaborators, non-technical stakeholders.

**Branding Terms:** Garvis, Pig Pen (LLM adapters), Pearl & Pig (overarching brand).

---

## Quick Links
- Demo UI: `http://localhost:3000` (or your deployment URL)
- Backend API Docs: `http://localhost:8000/docs` (if running locally)
- Audit Export: `http://localhost:8000/governance/audit/export?format=json`

---

## 0. Introduction (1 min)

"Welcome to Garvis! We're building an AI governance layer that ensures AI actions are truthful, auditable, and controllable. This PoC demonstrates how Garvis provides provenance and explainability, moving beyond black-box AI."

## 2. AKB Creation (2 mins)

*   "Garvis operates around Authoritative Knowledge Bundles (AKBs). Think of an AKB as a siloed, versioned dataset with its own rules."
*   "On this page, we’ll create a new AKB for a 'Marketing Plan Generation' task."
*   Enter AKB Name: `MarketingPlanGen`
*   Enter Owner: `MarketingOpsTeam`
*   Click **"Create AKB"**.
*   "See! Our AKB is created. All actions related to it are now being logged."

---

## 3. AI Adapter Selection (1 min)

*   "Garvis can orchestrate multiple AI models (the 'Pig Pen' concept). We can choose which AI to use, or even run them in parallel for verification."
*   "Here, we can select an adapter: `OpenAI-like LLM` (for advanced AI) or `LocalMock LLM` (for predictable, deterministic results)."
*   "For this demo, we'll use the default `OpenAI-like LLM`." (Or show switching if you want to highlight it).
*   Click **"Set Adapter"**.

---

## 4. Run Governance Demo: Generate Doc (3 mins)

*   "Now, we'll trigger a core governance workflow: generating a document based on this AKB."
*   Click **"Run Demo: Evaluate & Generate Document"**.
*   "Garvis first checks policies for this AKB. Then, it uses the selected AI adapter to create a document."
*   "Notice the system is designed with human-in-the-loop approval gates for sensitive actions like this." (If your PoC has the approval gate, demonstrate it. If not, explain it's a future/configurable feature).

---

## 5. Review Results: Provenance & Explainability (3 mins)

*   **(Show the generated document)** "Here's the document generated by the AI."
*   **(Point to Explainability Section)** "This is crucial: this isn't a black box. We see exactly *why* the AI generated this."
    *   "**Sources**: It tells us if it used data from this AKB, linked AKBs, or other external sources."
    *   "**Confidence**: A score showing how certain the AI is about the generated content."
    *   "**Rationale**: A brief explanation of the AI's reasoning."
*   "This verifiable output is key to building trust and ensuring AI adheres to our rules."

---

## 6. Audit Trail (2 mins)

*   "Every step from creation to generation is immutably logged."
*   "This audit log is critical for compliance and debugging."
*   **(Show Audit Export endpoint or a UI panel if added)** "We can export this log as JSON or CSV for regulators or internal review."
*   "This ensures 100% auditable truthfulness."

---

## 7. Cross-AKB Access Control (CAC) - Conceptual (1 min)

*   "Garvis enforces data isolation between AKBs. When one AKB needs data from another, it must be explicitly allowed via a CAC policy."
*   "If an unauthorized access attempt occurs, it's blocked and logged."
*   "This prevents AI hallucination and data leakage across different projects or clients."

---

*   "From here, we can build out advanced features: more templates (P&L, marketing plans), deeper integrations, and enterprise compliance tools."
*   "Garvis is designed to be layered, scalable, and adaptable to any business need, from solo users to large enterprises."
*   "Pearls & Pig are committed to building this future."
